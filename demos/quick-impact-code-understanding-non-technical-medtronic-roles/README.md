### Opening Hook (30 seconds)

"Code contains the truth about how your medical devices actually work - every design decision, every regulatory constraint, every customer workaround. But code is locked away from the 80% of people who need that knowledge but can't read Python or C++. Today I'll show you how non-technical roles can ask questions directly to your codebase - in plain English."

### Sequencing (Pick 3-4 for a 10-minute demo)

#### For Regulatory/Compliance Audience:
1. Start with Data Privacy Impact Analysis (GDPR/HIPAA relevance)
2. Add Requirements Traceability (FDA submission value)
3. Include Regulatory Change Impact (scoping updates)
4. Close with Legacy Code Forensics (safety rationale)

#### For Product/UX Audience:
1. Start with Legacy Code Forensics - Design Rationale (UX decisions)
2. Add Feature Usage Analysis (deprecation decisions)
3. Include Release Notes Preparation (customer communication)
4. Close with Business Logic Verification (refactoring safety)

#### For Support/Operations Audience:
1. Start with Bug Impact Assessment (incident response)
2. Add Expert Identification (knowledge risks)
3. Include Integration Data Flows (customer questions)
4. Close with Security Vulnerability Audit (risk management)

#### For Cross-Functional Leadership:
1. Start with Compliance Impact Analysis (regulatory)
2. Add Bug Impact Assessment (operations)
3. Include Legacy Code Forensics (product)
4. Close with Expert Identification (org risk)

## Metrics to Highlight

- **Knowledge Access:** "80% of people who need code knowledge can't read code"
- **Time Savings:** "Questions answered in seconds, not days of engineer time"
- **Risk Reduction:** "Prevent re-introduction of safety issues fixed years ago"
- **Compliance:** "FDA/GDPR audits require this level of traceability"
- **Institutional Knowledge:** "Code outlives people - preserve the 'why' not just the 'what'"

## Safety Tips

- Use real code search capability - very reliable
- Prepare specific file paths or module names in advance
- Have example questions written out for smooth delivery
- Know where good code comments exist for impactful reveals
- Can demo against public repos if internal code is sensitive
